        Sicherheitsscan
        
      

      

        
        

          

            

              
            

            

              

                

                  

                    pip-audit

                    Prüft alle installierten Python-Pakete im venv gegen bekannte CVEs
                    (NIST / PyPI Advisory Database). Findet verwundbare Abhängigkeiten.
                  

                  

                    Django Deploy Check

                    Führt manage.py check --deploy aus. Prüft Django-Sicherheitseinstellungen:
                    Cookie-Sicherheit, HSTS, Allowed Hosts, Debug-Modus u.m.
                    SSL-Redirect (W008) wird unterdrückt — wird von nginx übernommen.
                  

                  

                    HTTP / TLS Security Scan

                    Prüft Response-Header (HSTS, CSP, X-Frame-Options…), TLS-Version &
                    Zertifikat, Cookie-Flags sowie typische Konfigurationslecks (.env,
                    .git, Backups). Intern = direkter Gunicorn-Zugriff (kein TLS).
                    Extern = über HTTPS (Port 443) via nginx.
                  

                

                

                

                  
ENV = Änderung in der .env-Datei nötig

                  
Code = Änderung in settings.py nötig

                  
Installation = Server-/Paket-Problem
